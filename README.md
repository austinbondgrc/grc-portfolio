# 🛡️ Governance, Risk, and Compliance (GRC) Portfolio

Welcome to my practical GRC portfolio. 

As a Cybersecurity and GRC Professional transitioning into the German consulting market, I built this repository to demonstrate the practical application of international security standards. While certifications prove theoretical knowledge, this portfolio proves execution capability.

This repository contains a fictional but highly realistic Information Security Management System (ISMS) implementation for a hypothetical German SaaS provider ("MunichTech GmbH").

## 🎯 Core Frameworks Applied
*   **ISO/IEC 27001:2022** (Information Security Management Systems)
*   **NIS2 Directive** (EU Cybersecurity Legislation)
*   **BSI IT-Grundschutz** (German Federal standard mapping)

## 🗂️ Portfolio Contents

### 1. Governance & Scope (`/1_Governance_and_Scope`)
*   **ISMS Context & Scope Document:** Defines the boundaries of the ISMS, identifying internal and external stakeholders per Clause 4.
*   **Information Security Policy:** The high-level leadership directive outlining the security objectives (Clause 5).

### 2. Risk Management (`/2_Risk_Management`)
*   **Risk Assessment Methodology:** Outlines the qualitative approach used (Asset value x Threat likelihood x Vulnerability).
*   **The Risk Register (Excel):** A practical demonstration of identifying 5 critical business risks, calculating their impact, and detailing the mitigation strategy. *Note: Highlights the translation of technical vulnerabilities into financial exposure for executive review.*

### 3. Statement of Applicability (`/3_Statement_of_Applicability`)
*   **SoA Document (Excel):** A comprehensive mapping of the 93 Annex A controls (2022 revision), explicitly stating which controls are selected, the justification for inclusion/exclusion, and the implementation status.

### 4. Regulatory Compliance (`/4_Incident_Response_NIS2`)
*   **NIS2 Incident Reporting Workflow:** A flowchart and policy document outlining the mandatory 24-hour early warning and 72-hour incident notification process required by the directive.

### 5. Audit & Metrics (`/5_Audit_and_Metrics`)
*   **Internal Audit Checklist:** A structured evaluation tool (aligned with ISO 27001 Clause 9.2) used to assess the effectiveness and conformity of the implemented ISMS controls.
*   **Management Review Agenda:** A template outlining the required inputs and outputs for the annual executive review of the ISMS (Clause 9.3), ensuring continuous improvement and leadership visibility.
